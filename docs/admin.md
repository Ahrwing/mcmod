## 何为后台
MC百科后台内含有许多高级功能，如合并资料、批量修改分类，需要编辑员以上权限才可进入。

符合上述规则后，鼠标悬停于右上的用户名，左键单击`后台管理`即可。

## 模组管理

### 模组日志管理

此处可以查看负责模组的日志信息，同时对日志进行排序。

点击上方下拉栏选择要查看的模组，使用ID、主次要名或缩写来搜索。

#### 排序
排序方法有二

1. 自动排序：直接点击`名称`或者其他的表头，其下的分割线会加粗，双向箭头变为单向日志也会以箭头所示方向（例如向下即降序）自动排列日志。点击三次会切换回默认状态（双向箭头）。

>[danger] 多数时候自动排序无法正确排序，请在确认前检查排序效果是否符合预期，或点击多个表头对比效果。

![](https://cdn.jsdelivr.net/gh/mamaruo/Image/mcmodDoc/%E8%87%AA%E5%8A%A8%E6%8E%92%E5%BA%8F%E6%96%B020200821.gif)

2. 手动排序：长按要移动的日志条目，鼠标上下滑动进行拖动，放开鼠标完成移动。

![](http://bbs.mcmod.cn/data/attachment/album/201807/22/143529rodnpnvjvjkz385p.gif)

检查无误后，点击`确认排序`来保存排序。如果无异常，将会弹出`操作成功`的提示。如果不想保存，点击右上角的`战术刷新 (・ω・)`即可恢复原状。

## 资料管理

### 资料管理

此处可以查看负责 Mod 中的资料列表，并按类型、分类与名称筛选。还可批量执行修改分类、合并、综合、正文引用操作。点击名称可跳转至对应资料页，点击`操作`的按钮可快速编辑。

当资料列表中存在合并/综合父资料与子资料时，子资料前标注🔺，并在`名称`列标注`[合并至：(父资料图标)]`或`[综合至：(父资料图标)]`。

`不在“合并”与“综合”的父资料底下显示子资料`勾选后，不会显示任何子资料，除非其名称中含有搜索关键字。

[查看合并/综合指南](#合并)
>[info] 此处的资料是指网址为 /item/xxx.html 的所有页面。

#### 检索与查看
点击下拉菜单依次选择模组、类型、分类，即可查看对应分类中的资料。

使用搜索框来筛选资料，可使用主要名或次要名作为关键字。如果不选择特定的分类，将搜索整个类型；不选择类型则搜索整个模组。

<hr style="border:3 double" width="80%" align="left">
<details>
<summary>高级技巧（点击展开/收起）</summary>

一般的搜索将匹配名称与关键字相符，或名称中含有关键字的资料。

使用通配符来实现高级搜索：“\*”以表示任意字符串，包括空串；半角的“?”以表示一个字符。

例如：

搜索“\*石”会搜索出末地石、钻石；

搜索“\*石\*”会搜索出石砖墙、下界石英块、末地石；

搜索“??石”会搜索出铁矿石、煤矿石。

两种通配符可以同时使用实现更好的搜索效果。

想要将一堆资料结合起来，那么可以使`合并`和`综合`功能，二者的区别可以看下图：

![](https://admin.mcmod.cn/assets/img/diff.png)

合并是：有一堆子物品，那么选一个子物品并让其他的子物品结合到上面，这是合并；

综合是：有一堆子物品，编辑员随之添加一个父物品，那么把子物品结合到父物品上面，这是综合。

合并功能的介绍可以去[百科文档-资料添加/编辑教程-仅编辑员](http://help.qwq.fun/1835008#_67)处查看；下面会介绍综合功能。

综合功能上面已经解释了，是把子物品结合到父物品上，实际功能是：合并子物品到父物品上，在子物品的单页中引用父物品的介绍，在显示父物品分类的时候隐藏子物品和父物品的图标（看父物品单页的时候也会不显示图标，空内容的子物品不参与模组总填充率计算（父物品和有内容的子物品会算进）。

**所以综合物品之后，分类显示一个父物品的名字和超链接是很正常的，同时想要查看子物品，点击旁边的查看！子物品并没有消失！**

---

想要合并物品，首先应该选中想要合并上的物品，然后左上方就会显示`选择项合并至物品ID`，将选择的首物品的物品ID填入，然后点击`合并`按钮并等待合并完成。

注：选中功能目前有些问题，具体可见#525。

想要综合物品，首先应该添加一个父物品在子物品们的分类里，它不需要有材质，原名和译名应该随子物品的名字决定。

例：在“发电机”分类里有 基础蒸汽轮机(Basic Steam Turbine)、进阶蒸汽轮机(Advanced Steam Turbine) 和 涡轮蒸汽轮机(Turbo Steam Turbine)，那么应该添加一个物品名字叫蒸汽轮机(Steam Turbine)。

之后去后台查询这个父物品的ID，然后选中想要综合上的物品，左上方就会显示`选择项综合至物品ID`，将父物品的物品ID填入，然后点击`综合`按钮并等待综合完成。

想要批量更改一些物品的分类，选中想要更改分类的物品，然后选择分类并点击`更改`按钮并等待更改完成。

## GUI管理

此处可以查看、添加、编辑 编辑员负责区域 内所有Mod的GUI。

如果想要给自己负责的Mod添加GUI，点击左上处的`添加GUI`就会出现一个添加GUI的界面。首先应选择GUI所属的Mod，其次填入GUI的名称，这个名称会在合成表页面中部的`Mod设备`处显示，如图处：

![](http://bbs.mcmod.cn/data/attachment/album/201807/22/214130upzfnsnsix5ixc4r.png)

旁边的，GUI对应资料ID就是这个合成GUI的来源方块。例如原版的烧制GUI来自于熔炉，那么这里就相应的填“熔炉“这个物品在百科的ID；但例如沉浸工程的[HOP石墨粉](https://www.mcmod.cn/item/30915.html)必须要工业挤压机这个多方块结构才能合成，那么这里就要填写“工业挤压机”这个多方块结构的ID。ID可于[资料列表](#%E8%B5%84%E6%96%99%E5%88%97%E8%A1%A8)查看。之后点击左下的`选择文件`上传你所做的GUI图片。

然后再分别添加*材料*、*成品*、<em>燃料（随情况选择）</em>物品格子，并使用鼠标、`WSAD`键、`上下左右键`和`Shift键`将其移动到GUI图片上相应的格子上。具体效果如图：

![](http://bbs.mcmod.cn/data/attachment/album/201807/24/224023zv4ucd7i3nkdcqqv.png)

**要求的效果是：物品格子 对齐并严格贴合 GUI的格子，位置正确，最后是保证合成表使用时不出现问题。**

如果你添加格子的时候不小心添加错了一个物品格子**想删除**，那么就双击此物品格子，它就会被删除。如GIF所示：

![](http://bbs.mcmod.cn/data/attachment/album/201807/25/121100dxjz5cez1zv4yi4z.gif)

但是如果你GUI一次没有添加完**想隐藏格子**，那么就在右侧把没做完的格子的坐标改成(0,0)，此时**格子会保留，不会在 GUI管理的样式处和显示合成表时 显示，但会在添加/编辑合成表时显示**。具体效果如下列图：

![](http://bbs.mcmod.cn/data/attachment/album/201807/25/121824vdzz5oy6yddztmtv.png) ![](http://bbs.mcmod.cn/data/attachment/album/201807/25/121824tz19a3ligctniug6.png) ![](http://bbs.mcmod.cn/data/attachment/album/201807/25/121948i1bplfmp66fp7fpw.png) ![](http://bbs.mcmod.cn/data/attachment/album/201807/25/121824xkx53km663axo3sz.png)

注1：无论是材料、成品、燃料物品格子还是之后说到的能量条目，每个最多只能添加100个！但是目前无法在添加GUI时同时添加能量条目(见# 519)

注2：`上下左右键`或`WSAD键`只能把移动1像素，但按住Shift后就会移动10格。

当然对齐GUI上的格子是一个非常艰难的过程，此处提供一个对齐准确的方法：

1.将你要对其的（材料/成品/燃料）物品格子用鼠标大致移动到GUI所示格子处，如GIF所示：

![](http://bbs.mcmod.cn/data/attachment/album/201807/23/172405fgjuwjhj5wsag85w.gif)

2.之后使用键盘来 水平或竖直 移动物品格子，当移动露出了1列白条（即1像素）时，再移动1像素完成，即使 *物品格子的相应侧边* 与 *GUI的格子的白色侧边* 严格贴合。如GIF所示：

![](http://bbs.mcmod.cn/data/attachment/album/201807/24/233200i2uqagj17gjddab8.gif)

注：GUI上的这个白条基本上都是2像素的宽度，所以说如果那么再移动1像素即可。如果你的GUI不是的话，那么就继续移动，直到刚离开了白条（那时候也就是1像素），再移动回1像素即可。方法如GIF所示：

![](http://bbs.mcmod.cn/data/attachment/album/201807/25/091637p5nrfs34q3rx1c71.gif)

## 缓存管理
> 在编辑操作后系统会自动刷新对应的缓存，大多数情况下不需要手动刷新，频繁刷新会给系统带来压力。

——后台使用提示。


### 模组与资料缓存
此处可以刷新指定模组或资料的缓存。填写模组或资料 ID 后点击`刷新`即可。 

下方`按模组查看缓存信息`，点击下拉菜单，选择一个模组来查看。其大小取决于资料分类与资料的数量。

## 模组全局设置

此处可以设置管理模组的编辑规范。
1. 在论坛`百科编辑讨论`板块下的`编辑规范`中发帖。
2. 复制帖子地址。
3. 点击下拉栏选择对应的模组，粘贴链接。
4. 点击`更新`。

这样，编辑界面中的编辑帮助就会提示该模组有单独规范并提供跳转链接了。

## 资料列表挂载
挂载可以实现在同一个资料列表页面底下展示多个模组中的资料列表。适用于分包安装的模组，以及拓展多而杂的模组（如[拔刀剑]([https://www.mcmod.cn/item/list/366-1.html](https://www.mcmod.cn/item/list/366-1.html))）。

### 挂载管理
此处可以查看模组（❓权限存疑）的资料列表挂载信息，并可删除挂载列表。

### 添加挂载
此处可以添加模组文件列表中的挂载项。（❓权限存疑）

勾选默认折叠，被挂载的模组列表会在显示时折叠起来，需要浏览者点击展开。适用于资料较多或者挂载项较多的情况。挂载与被挂载的资料类型均可自由选择。

## 样式管理

此处可以自定义`模组内通用背景图片`与`资料列表表格`的样式。

输入 CSS 代码来编辑，点击`确认修改`来保存。

点击`来个例子`按钮后会自动生成一段带有注释的 CSS 代码，只需修改里面的数值即可完成样式调整，也可基于它进行更复杂的操作。

## 黑名单
 
编辑黑名单可以阻止指定用户对特定模组进行编辑，当用户多次违规且不听劝阻时，可将其加入编辑黑名单。

模组编辑黑名单只对非免审用户有效。

资深编辑员可将用户加入全站编辑黑名单。

### 模组编辑黑名单管理

此处可以查看负责模组的黑名单列表。
含有黑名单用户的`昵称`、`UID`、`被封时间`、`解封时间`、`执行人`、`封禁原因`，`昵称`和`执行人`两列的内容有超链接指向个人中心，并可在`昵称`一列执行解封操作。

除非主动点击`解封`，否则封禁信息在列表中不会移除。过了解封时间，用户的编辑限制会解除，同时`解封`按钮消失，列表中的信息将不可移除。

### 设置模组编辑黑名单

1.点击下拉栏选择封禁指定的模组。
2. 填写封禁用户的ID。
3. 指定封禁时长并填写封禁理由。
4. 点击`添加/更新黑名单`

在同一个模组中对同一名用户重复设置封禁信息会更新它的数据，封禁的开始时间会从现在重新算起（并非叠加）。


## 模组文件管理

### 上传文件

>[success] 文件管理页即将独立于新的服务器，与后台剥离，届时将不再有大文件上传时限，本分页也将不复存在。

>MC百科承诺：  
得到授权后将提供与官方一致的Mod文件下载，不篡改玩法，不署名，不附加恶意文件，不捆绑任何下载器或插件。

此处可以上传负责模组的文件以供浏览者下载。

- 选择Mod：在下拉列表中选择对应Mod，可搜索Mod ID(mid)，Mod中文名与Mod原名。
- 所属MC原版版本：输入Mod文件对应MC版本。支持多版本时，当支持某一大版本中所有版本(如1.10与1.10.2)，则填入"1.10.x"，当支持某一大版本中部分版本，则用"/"分开。(如1.12/1.12.1)
- 授权类型：授权通过某位百科用户与作者交流取得，即为 `手动授权` 。授权利用作者设置的开源协议，或作者主动提供授权，即为 `使用协议`。  

- 授权证明/说明楼：写明授权在[MOD下载区授权综合帖](https://bbs.mcmod.cn/thread-615-1-1.html)中的楼层链接。点击楼层号可直接复制楼层链接。
- 取得授权人(仅手动授权)：写明取得授权的用户ID，即个人中心末尾数字。

### 文件列表

>[success] 文件管理页即将独立于新的服务器，与后台剥离，届时将不再有大文件上传时限，本分页也将不复存在。

点击下拉式选单后可选择需要排序的Mod，默认按照Mod ID排序。搜索框中可通过Mod名称、原名、缩写、Mod ID来搜索需要的Mod。

点击表头可自动排序，无法保证正确性，排序后请复查。

拖动项目即可排序单个文件。(GIF参考[模组日志管理](#模组日志管理))
按住文件后可以通过 `鼠标滚轮`、`Page Up`、`Page Down`、`Home`、`End` 等方式来快速拖动文件至页首页尾等任何地方。

排序完成后需要点击末尾 `确认排序` 按钮，提交更改。

注意事项：

1. 文件排序规定为按照“MC版本 > 时间”的优先级排序，即先按照MC版本先后排序，再按照Mod文件更新时间排序。
2. `确认排序` 按钮目前至少需要一次排序操作才会显示。
3. 百科新上传文件不会置于第一位，而是默认第二位，因此每更新一个Mod的文件后都需要进行一次排序操作。

### 授权管理
此处可以修改负责模组的授权信息。

文件上传成功的前提是授权信息完整。

1. 点击下拉菜单选择模组。
2. 选择授权类型。
3. 填写授权证明（《[MOD下载区授权综合帖](https://bbs.mcmod.cn/forum.php?mod=viewthread&tid=615)》的楼层链接）。
4. 点击`确认修改`。

已填写授权信息的模组，在下拉菜单中的名称前会标记👌。
